# AgentRL 数据处理优化系统设计

## 1. 系统概述

### 1.1 目标
通过强化学习（PPO）自动学习最优的数据处理流水线配置，提升表格型时序数据的预测精度。

### 1.2 核心思路
将数据处理流水线建模为有序决策序列，每个处理步骤作为一个 Agent 决策点，RL 学习选择最优的算法组合。

## 3. 数据处理模块

### 3.1 各处理步骤及候选算法

| 步骤 | 候选算法 | 描述 |
|------|---------|------|
| 缺失值处理 | 删除/均值填充/中位数填充/前向填充/后向填充/插值/KNN填充/多重插补 | 处理缺失数据 |
| 异常处理 | Z-score/IQR/孤立森林/百分位法 | 检测并处理异常值 |
| 滤波降噪 | 中值滤波/均值滤波/高斯滤波/卡尔曼滤波/小波降噪 | 信号平滑 |
| 差分算子 | 一阶差分/二阶差分/季节性差分 | 时序特征提取 |
| 重采样 | 上采样/下采样/插值重采样/SMOTE | 平衡类别分布 |
| 标准化/归一化 | MinMax/Standard/Z-score/L2/RobustScaler | 特征缩放 |

## 4. 强化学习模块

### 4.1 状态空间 (State)
- 原始数据统计特征：均值、方差、缺失率、偏度、峰度、信噪比
- 当前处理后的数据特征
- 已选动作序列（历史决策）

### 4.2 动作空间 (Action)
- 每个处理步骤对应一个离散动作（选择具体算法）
- 动作空间大小 = 各步骤候选算法数量的乘积

### 4.3 奖励函数 (Reward)
```
reward = test_accuracy - baseline_accuracy
```
- baseline：未经过RL优化，使用默认配置处理数据的精度

### 4.4 PPO Agent
- 使用 Stable-Baselines3 库实现
- 网络结构：两层MLP（64, 64）
- 训练轮次：可配置，默认100轮

---

## 5. 预测模型模块

### 5.1 支持的模型
- **随机森林** (RandomForest)
- **LightGBM** (LGB)
- **XGBoost** (XGB)

### 5.2 模型配置
- 训练集/测试集划分：80% / 20%
- 使用默认超参数（RL优化数据处理，不优化模型超参数）

## 7. 数据流

1. **初始化**：加载数据集，定义数据处理流水线
2. **训练循环**：
   - Agent 选择每个步骤的处理算法
   - 执行数据处理流水线
   - 用处理后数据训练预测模型
   - 计算测试集精度作为奖励
   - PPO 更新策略网络
3. **输出**：最优数据处理配置 + 各模型精度提升结果
